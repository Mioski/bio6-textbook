# Анализ проекта: Интерактивный учебник биологии 6 класс

## 1. Исходные требования и интерпретация

Техническое задание предполагало создание интерактивного учебника для 6-го класса (ботаника) в форме Vue 3 SPA, который:
- Работает по протоколу `file://` (без локального сервера, двойной клик на `dist/index.html`)
- Охватывает все 62 темы УТК
- Имеет не менее 4 «глубоких» тем с полными симуляторами
- Мобильный дизайн (360–414 px, тёмная тема)
- Содержит не менее 20 иллюстративных элементов

**Ключевые архитектурные решения:**

1. **`createWebHashHistory`** вместо `createWebHistory` — единственный способ работы роутера по `file://`.
2. **`base: './'` в Vite** — позволяет относительно ссылаться на ресурсы в `dist/`.
3. **IIFE-формат сборки** (`format: 'iife'`, `inlineDynamicImports: true`) — вместо ES-модулей. Причина: `<script type="module" crossorigin>`, генерируемый Vite по умолчанию, блокируется в Firefox и Safari при открытии через `file://` из-за CORS. IIFE создаёт один JS-файл без module semantics — работает везде.
4. **JSON-driven контент** — все 10 файлов данных вынесены в `static/bio6/`, не в код компонентов. ContentService — единственная точка доступа к данным.
5. **Scoped styles** — все компоненты используют `<style scoped>`, никаких глобальных селекторов.
6. **`public/images/`** — Vite автоматически копирует `public/` в `dist/` при сборке. Все SVG-иллюстрации доступны как `dist/images/bio6/*.svg`.

### Интерактивные симуляторы

**Bio6CellSim** (Растительная клетка) — SVG 320×280 px с 7 кликабельными слоями (клеточная стенка → мембрана → цитоплазма → вакуоль → хлоропласт → митохондрия → ядро). Инфо-панель с названием, латынью и функцией.

**Bio6PhotosynthesisSim** (Фотосинтез) — три слайдера (свет 0–100%, CO₂ 0–100%, температура 0–40°C). Скорость = lightFactor × co2Factor × tempFactor × 100. Анимация пузырьков O₂ через CSS keyframes, определение лимитирующего фактора.

**Bio6SeedSim** (Строение семени) — SVG-схема семени фасоли. Кнопка «Разрезать» открывает 6 кликабельных зон с reveal-анимацией.

**Bio6LifeCycleSim** (Жизненный цикл) — круговая SVG-диаграмма с 6 фазами, навигация prev/next.

**Bio6GenericSim** (58 базовых тем) — диспетчер для 3 типов: `process_graph` (шаги процесса), `compare_two` (сравнение A и B), `tap_reveal` (сетка с открытием).

### Тренажёры

**Bio6SortTrainer** — бакет-сортировка: выбор элемента → выбор категории. Drag-free (tap-friendly).

**Bio6MatchTrainer** — задание на соответствие: кликабельные пары, подсветка результата.

**Bio6ReadingTask** — короткий текст + вопрос на понимание (4 варианта, объяснение ответа).

### Иллюстрации (20 уникальных SVG-файлов)

Все SVG создавались вручную с нуля, тёмная тема (#0d1117), подписи на русском языке:

### Нарративные цепочки (ChainService)

13 пар (prev → next) с recap/bridge плашками. Создают смысловую преемственность между темами (клетка → ткани → органы; корень → транспорт → транспирация и др.).

## 3. Ограничения реализации

1. **WebP-иллюстрации** (фотографии реальных объектов) — не созданы. Все иллюстрации в SVG (диаграммы/схемы), что соответствует требованию ТЗ для схем, но не покрывает фотографии живых растений.
2. **?v=2 версионирование** картинок — не реализовано (Vite добавляет хэш в имя бандла автоматически, для SVG не применимо).

## 4. Известные ошибки

На момент сдачи самопроверкой не обнаружено критических багов. Известные ограничения:
- При очень медленной загрузке (старые устройства) большой IIFE-бандл (~400KB) может задержать старт приложения.
- `localStorage` не синхронизируется между устройствами — прогресс ученика привязан к браузеру.

## 5. Архитектурные решения

**IIFE vs ES-модули**: Стандартный Vite-билд генерирует `<script type="module" crossorigin>`. Браузеры (особенно Firefox и Safari) блокируют загрузку cross-origin модулей при протоколе `file://`. Решение — `format: 'iife'` в rollupOptions + `inlineDynamicImports: true`. Это объединяет все компоненты в один файл (~400KB), устраняя проблему.

**Нет defineAsyncComponent**: Убраны ленивые загрузки симуляторов. При IIFE-формате dynamic import() несовместим с code splitting. Все 5 симуляторов загружаются сразу — приемлемо при размере бандла 400KB.

**provide/inject vs Vuex/Pinia**: Для линейного учебного SPA с одним предметоклассом provide/inject достаточно. Добавление Pinia усложнило бы архитектуру без реальной пользы.

**JSON в бандле**: Все JSON-данные включены в JavaScript-бандл при сборке. Это обеспечивает полный офлайн-режим без дополнительных HTTP-запросов.

## 7. Компромиссы

1. **IIFE vs code splitting** — один большой бандл вместо ленивой загрузки. Выбрано для совместимости с `file://`.
2. **SVG вместо WebP-фото** — все иллюстрации-схемы, нет фотографий реальных объектов. SVG идеально масштабируется и соответствует тёмной теме.
3. **localStorage** — прогресс только на устройстве. Для прототипа приемлемо.
