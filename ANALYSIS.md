# Анализ проекта: Интерактивный учебник биологии 6 класс

## 1. Исходные требования и интерпретация

Техническое задание предполагало создание интерактивного учебника для 6-го класса (ботаника) в форме Vue 3 SPA, который:
- Работает по протоколу `file://` (без сервера)
- Охватывает все 62 темы УТК
- Имеет 4 «глубоких» темы с полными симуляторами
- Мобильный дизайн (360–414 px, тёмная тема)

**Ключевые архитектурные решения:**

1. **`createWebHashHistory`** вместо `createWebHistory` — единственный способ работы роутера по `file://`.
2. **`base: './'` в Vite** — позволяет относительно ссылаться на ресурсы в `dist/`.
3. **JSON-driven контент** — все 9 файлов данных вынесены в `static/bio6/`, не в код компонентов. ContentService — единственная точка доступа к данным.
4. **Scoped styles** — все компоненты используют `<style scoped>`, никаких глобальных селекторов.

## 2. Архитектурные решения

### ContentService как единая точка доступа

Все 9 JSON-файлов импортируются исключительно через `ContentService`. Компоненты не импортируют JSON напрямую — это обеспечивает:
- Инверсию зависимостей (компоненты зависят от интерфейса, не от файловой системы)
- Возможность смены хранилища (JSON → API) без изменения компонентов

### ProgressService + localStorage

Прогресс хранится побайтово в `localStorage` как bitset пройденных элементов. Ключ `bio6_progress` содержит сериализованный объект `{nodeId: Set<elementId>}`. Процент вычисляется как `completed / total * 100`.

### ChainService: нарративные мостики

13 пар (prev_node → next_node) с краткими recap/bridge текстами. Встраивается в начало и конец страницы темы, создавая непрерывный нарратив.

### provide/inject вместо Vuex/Pinia

Для учебного SPA с линейной структурой передача сервисов через `provide()` в `bio6-base.vue` достаточна и не требует внешнего стора. Компоненты получают сервисы через `inject()`.

## 4. Интерактивные симуляторы

### Bio6CellSim (plant_cell)
SVG 320×280 px с 7 наложенными слоями (wall → membrane → cytoplasm → vacuole → chloroplast → mitochondria → nucleus). Клик по слою показывает инфо-панель с названием, латынью и функцией. Легенда со всеми элементами.

### Bio6PhotosynthesisSim (photosynthesis)
Три `<input type="range">`: свет (0–100%), CO₂ (0–100%), температура (0–40°C). Скорость = `lightFactor × co2Factor × tempFactor × 100`. Анимация пузырьков O₂ через CSS keyframes. Определение лимитирующего фактора и текстовый инсайт.

### Bio6SeedSim (seed_structure)
SVG-схема семени фасоли 300×260 px. Кнопка «Разрезать» открывает внутренние части. 6 кликабельных зон с reveal-анимацией. Панель описания снизу.

### Bio6LifeCycleSim (plant_life_cycle)
Круговая SVG-диаграмма 300×300 px с 6 фазами (seed → germination → seedling → growth → flowering → pollination). Навигация prev/next, индикатор текущей фазы, описание фазы.

### Bio6GenericSim (базовые темы)
Диспетчер для 3 типов:
- `compare-two` (GenericCompareTwo) — переключатель «А/Б», факты
- `tap-reveal` (GenericTapReveal) — сетка карточек, клик открывает описание
- `process-graph` (GenericProcessGraph) — вертикальный граф шагов

## 5. Тренажёры

### Bio6SortTrainer
Бакет-сортировка: пользователь выбирает элемент из списка, затем бакет. Drag-free (tap-friendly для мобильных). Кнопка «Проверить» показывает результат с объяснениями ошибок.

### Bio6MatchTrainer
Задание на соответствие: левая колонка фиксирована, правая — перемешана. Клик по левому → клик по правому создаёт пару. Подсветка правильных/неправильных пар после проверки.

## 6. Навигационная логика

```
/bio6/index → список блоков
/bio6/block/:blockId → список тем блока
/bio6/node/:nodeId → полная страница темы
```

Хэш-роутинг обеспечивает совместимость с `file://`. NavigationService обёртывает вызовы роутера, decoupling компонентов от прямой зависимости от `useRouter()`.

## 8. Компромиссы и ограничения

1. **Иллюстрации**: SVG-схемы созданы для 4 глубоких тем; растровые иллюстрации не создавались (по решению заказчика).
2. **Офлайн-режим**: JSON импортируются в бандл при сборке — контент доступен без интернета.
3. **Без серверного рендеринга**: SPA с хэш-роутингом, SEO не критично для учебного приложения.
4. **Локальное хранение прогресса**: `localStorage` — не синхронизируется между устройствами, что приемлемо для прототипа.
